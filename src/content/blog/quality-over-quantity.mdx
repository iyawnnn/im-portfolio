Ten repositories. Zero usable applications.

I looked at my GitHub profile a few months ago and realized something terrible. I had spent a year building projects, but I would not want a real user to touch any of them.

When I first started doing school assignments and side projects, the only goal was to finish. I just wanted the code to compile. If the login button worked, I closed my laptop and moved on. I was obsessed with quantity. I wanted to build as many things as possible to make my resume look packed.

I layered logic on top of logic. I tried to be clever instead of being clear.

But then I actually tried to use one of my own applications a month later. 

The UI was clunky. The security was basically nonexistent. The user experience was completely ignored. It hit me that a portfolio with ten broken apps is much worse than a portfolio with two highly polished products.

So I stopped building new things. I went back to my old code.

I opened up [Kodasync](/projects/kodasync) and [SubVantage](/projects/subvantage). Instead of adding more features, I started deleting code. I stripped out the chaotic designs and focused entirely on the core mechanics. 

I spent days just refining the minimalist UI to make it feel like a premium developer tool. I researched common vulnerabilities and patched the security holes. I stopped trying to outsmart the frameworks and just let the native caching and routing do the heavy lifting.

It worked immediately. 

Somewhere between deleting my old messy code and deploying the clean versions, something clicked:

> You will never build a perfect system. But you can always build a system that respects the user.

Code is never truly finished. There will always be bugs. But taking the time to write clean, secure, and beautifully designed applications changes how you think as an engineer. 

The days of rushing to the finish line just to check a box are over. It is time to just build things that actually work well.
